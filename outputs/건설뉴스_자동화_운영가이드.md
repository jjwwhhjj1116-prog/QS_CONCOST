# CONCOST 건설 기회정보 레이더 운영 가이드

## 지금 구현된 수집 흐름

1. 입찰공고: 나라장터·LH·한국도로공사에서 실제 공고를 수집합니다.
2. 건설 주요뉴스: 조달청·국토교통부 공식 게시물을 읽고 건설·건축·주택·정비사업·안전진단·공사비 관련 제목만 선별합니다.
3. 법규·제도 개정: 국가법령정보 OPEN API에서 최근 공포 법령을 조회하고 건설 관련 법령만 선별합니다.
4. 정규화: 출처, 고유번호, 제목, 날짜, 원문 링크, 요약을 공통 형식으로 변환합니다.
5. 중복 제거: `출처 + 원문 고유번호`가 같은 항목은 새 글을 만들지 않고 갱신합니다.
6. 관련도 점수: 공사비·원가·견적·안전진단·재건축·재개발·건설산업 등 CONCOST 업무 키워드에 가중치를 줍니다.
7. 자동 게시: 선별된 항목은 SQLite에 저장되는 즉시 웹사이트의 `건설 주요뉴스` 또는 `법규·제도 개정` 탭에 표시됩니다.

산군의 화면과 기사를 복제하는 구조가 아닙니다. 산군과 비슷한 업무 카테고리를 제공하되, 원문은 공식 출처에서 수집하고 각 카드가 원문으로 연결되게 했습니다. 산군 개발자센터 API가 승인되면 별도 공식 커넥터로 추가하는 방식이 안전합니다.

## API 인증값 보안

- 국가법령정보 인증값은 브라우저 저장소나 HTML/JavaScript에 기록하지 않습니다.
- 관리자 로그인 후 입력한 값은 Windows DPAPI로 암호화해 SQLite에 저장합니다.
- 암호문은 해당 Windows 사용자 계정에서만 해독할 수 있습니다.
- 국가법령정보 인증값은 저장 후 웹 화면으로 다시 반환하지 않고 `설정됨/미설정` 상태만 표시합니다.
- 로그와 뉴스 데이터에는 인증값을 남기지 않습니다.
- DB와 Windows 계정이 함께 유출되지 않는 한 인증값을 바로 사용할 수 없습니다.

주의: 다른 Windows 계정으로 예약 작업을 실행하면 DPAPI 인증값을 해독할 수 없습니다. 반드시 대시보드를 실행하는 동일 계정으로 예약 작업을 구성해야 합니다.

## 매시간 자동 수집 권장 구성

Windows 작업 스케줄러에서 다음과 같이 등록합니다.

1. 시작 메뉴에서 `작업 스케줄러`를 엽니다.
2. `기본 작업 만들기`를 선택하고 이름을 `CONCOST 건설정보 수집`으로 지정합니다.
3. 트리거는 `매일`, 시작 시간은 오전 7시로 설정합니다.
4. 고급 설정에서 `1시간마다 반복`, 기간은 `1일`로 지정합니다.
5. 동작은 `프로그램 시작`을 선택합니다.
6. 프로그램에는 이 프로젝트의 `collect_now.cmd` 절대경로를 지정합니다.
7. `사용자가 로그온할 때만 실행`을 선택하고 대시보드와 같은 Windows 계정을 사용합니다.
8. 등록 후 작업을 한 번 수동 실행해 `건설 주요뉴스`와 `법규·제도 개정` 탭에 새 항목이 나타나는지 확인합니다.

권장 주기는 입찰 30~60분, 뉴스 2~4시간, 법령 하루 1~2회입니다. 현재 단일 수집 명령은 세 종류를 함께 확인하므로 초기 MVP에서는 1시간 주기가 가장 단순합니다.

## 매일 알림 자동화의 다음 단계

현재 `오늘 요약 미리보기`가 제목·점수·링크를 묶어 줍니다. 실제 전송에는 다음 중 하나를 붙이면 됩니다.

- 이메일: 회사 SMTP 또는 Microsoft 365 SMTP/API를 연결해 매일 오전 8시 30분에 상위 점수 항목을 발송합니다.
- 카카오워크: 회사 카카오워크 Webhook이 있으면 요약 메시지를 채널로 전송합니다.
- 카카오톡: 일반 개인 카카오톡 자동 발송은 제약이 많으므로 카카오 비즈메시지/알림톡 사업자 연동이 필요합니다.

발송 전에는 `신규 항목만`, `40점 이상`, `같은 원문은 한 번만`이라는 세 조건을 적용하는 것이 좋습니다. 초기 1~2주는 자동 게시 + 관리자 미리보기로 오탐을 확인한 뒤 외부 발송을 켜는 방식이 안전합니다.

## 장애 확인 순서

1. `run_dashboard.cmd`를 더블클릭합니다. 파일 내부에서 8765 포트의 이전 서버를 정리하고 최신 서버를 시작합니다.
2. 관리자 설정에서 각 API가 설정됨으로 표시되는지 확인합니다.
3. `지금 전체 수집`을 눌러 출처별 성공/실패 문구를 확인합니다.
4. 국가법령정보가 실패하면 승인 상태, 인증값, 동일 Windows 계정 실행 여부를 확인합니다.
5. 출처 사이트 HTML이 바뀌면 뉴스 수집 파서를 갱신해야 합니다. 공식 API가 제공되는 출처는 HTML 수집보다 API를 우선합니다.
